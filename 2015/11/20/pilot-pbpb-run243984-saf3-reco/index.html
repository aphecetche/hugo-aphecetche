SAF3 reconstruction of "pilot" PbPb 2015 run 243984

Fri, Nov 20, 2015

This run was reconstructed as part of the regular LHC15m muon_calo_pass1, but as such used the pp recoparams, where the roads are way too large for PbPb and so basically all central events were killed during reconstruction. Which can be clearly see in Diego’s presentation at the 2015, 18th nov QA meeting, e.g. the number of tracker tracks per CMSL trigger is too low (3).

In order to cross-check that the intended recoparams (actually the ones from LHC13def or LHC11h) were doing better, I did a SAF3 reconstruction of this run only (270 chunks).
